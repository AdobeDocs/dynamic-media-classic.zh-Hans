# 有关从升级的常见问题解答 [!DNL Dynamic Media Classic] to [!DNL Dynamic Media]

## 一般信息

+++**什么是 [!DNL Dynamic Media] in [!DNL Adobe Experience Manager] 资产？**
[!DNL Dynamic Media] 是 [!DNL Adobe Dynamic Media Classic] (以前称为Scene7) [!DNL Experience Manager] 资产。 该解决方案将资产管理的强大功能与富媒体交付结合在一起。 功能包括：

* 用于管理图像和视频的单个用户界面和平台。
* 创新的推销功能。
* Adobe强大而经验证的交付平台。
* 与 [!DNL Experience Manager] 资产。

+++

+++**升级到 [!DNL Dynamic Media]?**

* 协作和共享文件同步 [!DNL Adobe Creative Cloud] 应用程序。
* 企业级数字资产管理，具有强大的元数据支持、智能搜索、Lightbox和收藏集、版本控制以及供供应商、合作伙伴和特许商使用的安全资产共享。在制品资产的审核和批准工作流程。
* 易于采用和使用新的UI。
* 使用可促进转化的图像和视频创建购物/交互式媒体体验，并提高用户参与度和满意度。
* 将促销活动资产与产品信息结合使用，以简化购物车点击过程。
* 使用WYSIWYG查看器设计器轻松创建、调整、品牌和部署交互式查看器。
* 将优化的富媒体交付到 [!DNL Experience Cloud] 解决方案。
* 与集成 [!DNL Experience Cloud] 用于跨营销接触点进行高级资产分析、定位和资产重用。 这些接触点包括 [!DNL Adobe Campaign] 对于电子邮件， [!DNL Adobe Social] 用于社交渠道和 [!DNL Experience Manager] 适用于响应式Web和移动设备应用程序的网站。

+++

+++**是 [!DNL Dynamic Media] 是否使用现有AdobeCDN（内容交付网络）？**
是的， [!DNL Dynamic Media] 使用Adobe强大的顶层交付网络。

* Internet Retail 1000的顶级富媒体供应商，连续9年。
* 24/7/265支持，99.95%的SLA。
* 经验证的基础架构为全球范围的800多个客户端提供服务，每月3.5 PB的流量，管理资产超过5亿个，流量同比增长60%。

+++

+++**什么是 [!DNL Dynamic Media Classic]? Adobe是否在更改Adobe Scene7的名称？**
Adobe将Adobe Scene7的名称更改为 [!DNL Dynamic Media Classic].

+++

## 升级过程和工具

+++**谁有资格参加升级计划？**
当前 [!DNL Dynamic Media Classic] (前称Scene7) [!DNL Experience Manager].

+++

+++**如何开始升级过程？**
联系您的AdobeCSM（客户服务管理）代表或 [电子邮件s7support@adobe.com](mailto:s7support@adobe.com) 主题行 `[!DNL Dynamic Media] Upgrade Program`.

+++

+++**如何处理升级过程？**
升级由AGS(Adobe全球服务)处理，并视为服务项目。 Adobe仅提供资产的迁移。 客户、AGS或合作伙伴负责所有其他升级方面和步骤。 在汇总级别，升级计划包含以下内容：

* 配置公司/用户帐户。
* 从复制资产 [!DNL Dynamic Media Classic] (前称Scene7) [!DNL Dynamic Media] 组件 [!DNL Experience Manager] 资产(由Adobe通过自动升级工具提供)。
* 配置成像和视频的设置。
* 修改生产流程并培训用户。

+++

+++**升级过程需要多长时间？**
升级过程时间因多个因素而异，包括但不限于：资产数量和资产大小。 AGS或合作伙伴可管理项目时间轴。

+++

+++**如何检查升级状态？**
升级过程时间因多个因素而异，包括但不限于：资产数量和资产大小。 AGS或合作伙伴可管理项目时间轴。

+++

+++**升级到 [!DNL Dynamic Media]?**
升级过程时间因多个因素而异，包括但不限于：资产数量和资产大小。 AGS或合作伙伴可管理项目时间轴。

+++

+++**许可成本有不同吗？**
与您的Adobe客户成功经理代表合作，以了解有关定价的详细信息。

+++

+++**升级过程是否存在停机？**
否. [!DNL Dynamic Media Classic] 在升级过程中继续不间断工作。 升级完成并验证内容后，所有用户都只能在 [!DNL Dynamic Media] 组件 [!DNL Experience Manager] 资产。

+++

+++**是否需要升级就绪核对清单中的所有步骤？**
否. 的 [准备清单](/help/upgrade-readiness.md) 包含必需和可选的最佳实践步骤。

+++

+++**是否需要升级？**
否. Adobe继续全面支持和维护 [!DNL Dynamic Media Classic] （客户请求的错误修复、安全修复、平台可扩展性和可靠性）。

当您准备好利用 [!DNL Dynamic Media].

+++

+++**我能否继续使用 [!DNL Dynamic Media Classic] (以前称为Adobe Scene7) [!DNL Dynamic Media]?**
升级到 [!DNL Dynamic Media]，则只应将Dynamic Media用于成像和视频。 您可以继续使用 [!DNL Dynamic Media Classic] 仅适用于 [!DNL Dynamic Media] 包括：

* 可视配置器（图像作者、图像渲染）。
* 图像模板。
* eCatalog.

+++

+++**Adobe提供了哪些工具来自动执行升级过程？**
在初次启动升级计划时，Adobe提供了自动从 [!DNL Dynamic Media Classic] to [!DNL Dynamic Media] in [!DNL Experience Manager] 资产。

+++

+++**执行现有操作 [!DNL Dynamic Media Classic] 升级期间和之后，URL、API集成和查看器是否会继续工作？**&#x200B;是的。您可以继续使用 [!DNL Dynamic Media Classic] (以前称为Adobe Scene7)按原样发布和交付基础结构 [!DNL Dynamic Media] 解决方案。

+++

+++**是否必须更新生产URL?**
否. Adobe继续使用 [!DNL Dynamic Media Classic] 按原样发布和交付基础结构 [!DNL Dynamic Media] 解决方案。 此方法的好处是您不必更改网页上的任何生产URL，从而最大限度地减少迁移到 [!DNL Dynamic Media].

+++

+++**我是否必须重写API集成和其他自动化脚本？**
否. Adobe继续使用 [!DNL Dynamic Media Classic] 按原样发布和交付基础结构 [!DNL Dynamic Media] 解决方案。 此外，所有资产都会复制到 [!DNL Dynamic Media Classic]. 此方法的好处是您不必重写任何基于API的集成或自动脚本，从而最大限度地减少迁移到的风险和工作 [!DNL Dynamic Media].

+++

+++**我是否必须进行更改或重新开发自定义查看器？**
否. Adobe继续使用 [!DNL Dynamic Media Classic] 按原样发布和交付基础结构 [!DNL Dynamic Media] 解决方案。 此方法的好处是您可以继续使用自定义的查看器，从而最大限度地减少迁移到的风险和工作 [!DNL Dynamic Media].
+++

+++**如何将我的设置（如图像预设、视频编码）迁移到 [!DNL Dynamic Media]?**
必须在 [!DNL Dynamic Media]. 作为服务项目的一部分，Adobe全球服务或合作伙伴可以提供协助。

+++

+++**如何设置用户和权限（SSO或LDAP选项）？**
必须在 [!DNL Dynamic Media]. 作为服务项目的一部分，AGS或合作伙伴可以提供协助。 但是，升级到 [!DNL Dynamic Media] 提供SSO/LDAP集成，从而提高用户管理效率。 此外， [!DNL Dynamic Media] 提供了强大的角色和权限来控制用户访问权限。

+++

+++**我是否仍可以使用FTP批量/批量上传资产？**&#x200B;是的。您无需修改现有的工作流即可摄取，并可以继续计划基于FTP的上传。

+++

+++**哪些资源可用于培训新用户？**
通过ADLS(Adobe数字学习服务)提供培训。 [!DNL Dynamic Media] 两门课程的功能包括：管理和交付数字资产以及自定义数字资产。

+++

+++**是 [!DNL Dynamic Media] 适用于所有地理区域？**&#x200B;是的。Adobe在北美、欧洲和亚太地区设有数据中心。

+++

+++**多久 [!DNL Dynamic Media Classic] 作为独立产品存在吗？**
Adobe继续全面支持和维护 [!DNL Dynamic Media Classic] （客户请求的错误修复、安全修复、平台可扩展性和可靠性）。

+++

+++**存储容量 [!DNL Dynamic Media]?**
包含60GB的存储空间。 您可以购买250 GB的块中的额外存储。 检查您的合同详细信息，以获取您当前的存储配额。

+++

+++**用于测量的量度 [!DNL Dynamic Media] 使用情况？**
每月页面查看次数(PVM)。 “页面查看”是指电子邮件或Internet网站网页的单次查看。 它还包括应用程序屏幕视图、应用程序屏幕状态、移动网页和社交网页。 每次加载或刷新网页、加载应用程序或通过打开或查看的电子邮件呈现或显示内容时，都会发生页面查看次数。

+++

+++**个性化媒体何时将在 [!DNL Dynamic Media]?**
Adobe正积极努力将个性化媒体功能添加到 [!DNL Dynamic Media]. 即将提供有关发布时间的更多信息。

+++

+++**在升级过程中如何验证资产？ 是否需要我进行手动验证？**
Adobe会对移动到的所有资产执行自动验证 [!DNL Dynamic Media]. 对关键站点、页面和体验进行常规的手动验证，验证由 [!DNL Dynamic Media] 。

+++

+++**是否有 [!DNL Dynamic Media]?**&#x200B;是的。有关详细信息，请联系您的Adobe客户成功经理代表。

+++

+++**我能否在 [!DNL Dynamic Media]?**&#x200B;是的。您可以在 [!DNL Dynamic Media].

+++

+++**是 [!DNL Dynamic Media] 是否具有升级所需的功能？ 提供了哪些功能 [!DNL Dynamic Media] on [!DNL Experience Manager] 资产？**
请参阅 [功能比较](/help/upgrade-feature-comparison.md) 页面以了解更多信息。

+++

+++**哪些功能仍仅在 [!DNL Dynamic Media Classic]? 我还能升级吗？**
需要满足以下条件的客户可以继续访问 [!DNL Dynamic Media Classic] 强大功能：

* 可视配置器（图像作者、图像渲染）。
* 图像模板。
* eCatalog.

另请参阅 [功能比较](/help/upgrade-feature-comparison.md) 页面以了解更多信息。

+++

+++**正在发生的情况 [!DNL Dynamic Media Classic] Media Portal解决方案？**
[!DNL Experience Manager] Brand Portal是 [!DNL Dynamic Media Classic] 媒体门户。
